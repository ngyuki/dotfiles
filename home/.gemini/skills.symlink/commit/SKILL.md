# Git Commit Skill

このスキルは、Git でステージングされている変更をコミットするのに役立ちます。

## 主な機能

- ステージングされた変更 (`git diff --cached`) と最近のコミット履歴 (`git log`) を分析し、コミットメッセージを提案します。
- ステージングされたファイルがない場合は、ワーキングディレクトリの変更をステージングするかどうかを尋ねます。
- 生成されたメッセージを使ってコミットを実行します。
- コミットフックでエラーが発生した場合の再試行オプションを提供します。

## ワークフロー

1.  **変更の確認:**
    - `git status` と `git diff --cached` を実行して、ステージングされている変更を確認します。
    - ステージングされているファイルがない場合:
        - `git diff` を実行して、ワーキングディレクトリに変更があるか確認します。
        - 変更がある場合は、どのファイルをステージングするかをユーザーに尋ねます。
        - 変更がない場合は、その旨をユーザーに伝えて終了します。

2.  **コミットメッセージの生成:**
    - `git diff --cached` の内容と `git log -n 3` の出力を分析します。
    - プロジェクトでコミットメッセージに関する規約があれば、それに従います。
    - 分析結果に基づいてコミットメッセージを作成し、ユーザーに提案します。

3.  **コミットの実行:**
    - ユーザーがメッセージを承認したら、それを一時ファイル (`{{project_temp_dir}}/commit_message.txt`) に書き込みます。
    - `git commit -F {{project_temp_dir}}/commit_message.txt` を実行してコミットします。
    - `git commit -m` を直接使用すると、メッセージに特殊文字が含まれる場合に問題が発生する可能性があるため、この方法を推奨します。

4.  **エラーハンドリング:**
    - コミットが pre-commit フックなどで失敗した場合、`-n` (または `--no-verify`) オプションを付けてコミットを再試行するかどうかをユーザーに尋ねます。
    - ユーザーが同意した場合は、`git commit -F {{project_temp_dir}}/commit_message.txt -n` を実行します。
