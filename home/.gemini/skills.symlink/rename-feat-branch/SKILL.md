# Rename Feature Branch Skill

このスキルは、現在の Git ブランチを、そのブランチで行われた変更内容に基づいて適切な機能ブランチ名にリネームするのに役立ちます。

## 主な機能

- upstream ブランチとの差分 (`git log '@{u}'...@ -p`) を分析し、変更内容に最も適したブランチ名を提案します。
- プロジェクトの既存のブランチ命名規則 (`git branch --list`, `git log`) を考慮に入れます。
- 提案したブランチ名でリネームを実行する前に、必ずユーザーの確認を求めます。

## ワークフロー

1.  **変更内容の分析:**
    - `git log '@{u}'...'@' -p` コマンドを実行して、現在のブランチに加えられたコミットと差分を詳細に分析します。
    - この情報から、ブランチの目的（新機能、バグ修正、リファクタリングなど）を把握します。

2.  **命名規則の推測:**
    - `git log -n 20 --oneline --graph` を実行して、最近のコミットメッセージの傾向を把握します。
    - `git branch --list` を実行して、既存のブランチ名のパターン（例: `feat/...`, `fix/...`）を調査します。
    - プロジェクトでブランチ命名規則があれば、それを最優先します。

3.  **ブランチ名の提案:**
    - 分析結果と命名規則に基づいて、新しい機能ブランチ名を生成します。
    - 生成したブランチ名をユーザーに提案し、リネームを実行してよいか確認します。

4.  **ブランチのリネーム:**
    - ユーザーが提案を承認した場合、`git branch --move <新しいブランチ名>` を実行して、現在のブランチをリネームします。
    - ユーザーが提案を拒否した場合は、処理を中断します。
