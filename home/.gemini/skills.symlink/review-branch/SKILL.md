# Review Branch for Pull Request

このスキルは、プルリクエストを作成する前の準備として、アップストリームブランチとの差分、コミット履歴、ブランチ名をレビューし、プロジェクトの規約に準拠しているかを確認します。

## ワークフロー

1.  **アップストリームブランチの特定**:
    *   `git rev-parse --abbrev-ref --symbolic-full-name '@{u}'` を実行して、アップストリームブランチ名（例: `origin/dev`）を特定します。
    *   もしエラーになる場合（アップストリームが未設定など）、ユーザーにマージ先のブランチ名を尋ねます。

2.  **リモート情報の更新**:
    *   `git fetch --all` を実行して、すべてのリモートリポジトリの最新情報を取得します。

3.  **ブランチ名の規約チェック**:
    *   `git branch --show-current` を実行して、現在のブランチ名を取得します。
    *   ブランチ名の命名規則（例: `feat/`, `bugfix/` など）に関する記述を探します。
    *   現在のブランチ名がその規則に準拠しているかチェックします。
    *   **もし準拠していない場合**: ユーザーに規約違反を伝え、適切なブランチ名への変更を提案します。

4.  **コミット履歴のレビュー**:
    *   `git log '@{u}'..HEAD` を実行して、アップストリームとの差分があるコミットのログを取得します。
    *   各コミットについて、以下の点をレビューします。
        *   **コミットメッセージ**: プロジェクトの規約（例: Conventional Commits）に準拠しているか？
        *   **コミットの粒度**: 1つのコミットが単一の関心事に集中しているか？大きすぎたり、無関係な変更が混じっていないか？
    *   `git show COMMIT_HASH` を実行して、各コミットの差分を個別に確認します。

5.  **ブランチ全体の差分レビュー**:
    *   `git diff --name-status '@{u}'...` を実行し、変更されたファイルのリストを取得します。
    *   変更された各ファイルについて、`git diff '@{u}'... FILE_PATH` を実行して詳細な差分を取得します。
    *   `references/review-checklist.md` のチェックリストに基づいてコードレビューを実施します。

6.  **レビュー結果の報告**:
    *   レビュー結果（ブランチ名、コミット履歴、コード全体）を整形し、問題点、懸念点、改善提案をリストアップしてユーザーに報告します。
    *   「これらの点を修正しますか？」と問いかけ、ユーザーの次の指示を待ちます。

7.  **レビュー結果の詳細報告**:
    *   ユーザーから詳細な説明を求められた場合、以下の点について掘り下げて報告します。
        *   **変更の背景**: なぜこの変更が必要だったのか？解決しようとしている課題は何か？
        *   **コードの変更点（詳細）**:
            *   主要な変更箇所を特定し、そのファイルや関数がどんな役割を持っているかを説明します。
            *   変更前と変更後のコードを比較し、どのようにロジックが変わったのかを具体的に解説します。
            *   新しい関数やクラスが追加された場合は、その目的と動作を説明します。
        *   **テストの妥A当性**:
            *   追加・修正されたテストが、新しい機能や修正点をどのようにカバーしているかを説明します。
            *   テストケースが適切かどうか（エッジケースを考慮しているかなど）をレビューします。
        *   **全体的な評価**:
            *   変更が一貫しており、プロジェクトの設計思想に合っているかを評価します。
            *   今後の改善点や、他に影響がありそうな箇所があれば指摘します。

## 注意事項

*   ブランチのリネームやコミットの修正（rebaseなど）は、ユーザーの明確な許可があった場合にのみ提案・実行してください。
