# Humanizer JA 🇯🇵

AIくさい日本語を、人間が書いた文章に変換するClaude Codeスキル。

## これは何？

Claude、ChatGPT、Gemini等が生成した日本語テキストには、特有の「AIっぽさ」があります。「浮き彫りにしており」「今後の展開が注目されます」「**速度:** 高速化」——こういった表現は、人間はまず使いません。

このスキルは、そういったAIパターンを20項目に分類し、Claude Codeが文章を生成する時点で自動的に回避するようにするものです。

## インストール

```bash
# Claude Codeのスキルディレクトリにコピー
cp -r humanizer-ja ~/.claude/skills/humanizer-ja
```

## 使い方

Claude Codeで文章を生成した後:

```
この文章を人間っぽくして
```

または、文章生成時に最初から指定:

```
humanizer-jaスキルを使って、〇〇について記事を書いて
```

## 何をチェックするか（20パターン）

| カテゴリ | パターン |
|---------|---------|
| **語彙・表現** | 意義の過剰強調、定型評価語、カタカナ語の過剰使用、曖昧な出典 |
| **構造・フォーマット** | 太字+コロン箇条書き、三点セット強制、全角ダッシュ乱用、見出しの過剰構造化 |
| **文体・トーン** | 説教くさい前置き、接続詞の多用、否定並列構文、追従的トーン、-ing的付け足し |
| **日本語固有** | 敬語の均一化、主語の過剰明示、「することができます」、体温のない結論 |
| **セルフ監査** | 冒頭チェック、結論チェック、二重チェック |

## blader/humanizer との違い

[blader/humanizer](https://github.com/blader/humanizer)（英語版、10,000+ stars）にインスパイアされていますが、以下が異なります:

- **日本語特化**: 敬語の均一化、主語の過剰明示、「することができます」等の日本語固有パターンを追加
- **実践寄り**: 学術的な分析より「今すぐ使えるチェックリスト」を重視
- **声の入れ方**: パターン除去だけでなく「消した後に何を入れるか」まで具体的に指示
